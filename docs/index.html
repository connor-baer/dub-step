DubStep

DubStep

Many UI widgets have steps with boundaries and a "current" index. dub-step provides an API for updating an index.

  • Directly when passed to getIndexControlProps(index).
  • Incrementally when getNextControlProps/getPreviousControlProps is applied to a button.
  • On swipe when getSlideProps is applied to a container of "slides".
  • On a timer when getPlayControlProps/getPauseControlProps are applied to a button.

The props provided affect how/when the index is updated and modify the above behaviors.

StateAndHelpers

The state of dub-step and prop getters/actions for changing the state are exposed as a parameter to the render callback.

The paramters of this function can be split into three categories: State, Prop getters, and Actions.

  • State: State properties of dub-step exposed to your render code. Controlled state can be passed as a prop and "controlled" by an outside component/router/store.
  • Prop getters: Functions named like get*ControlProps. They are used to apply props to the elements that you render. This gives you maximum flexibility to render what, when, and wherever you like. You call these on the element in question (for example: <button {...getNextControlProps()}))>Next</button>. It's advisable to pass all your props to that function rather than applying them on the element yourself to avoid your props being overridden (or overriding the props returned).
  • Actions: Call these to directly change the state of dub-step.
